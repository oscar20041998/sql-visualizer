### 4. AST Extraction Functions

#### `extractTablesFromAST(ast)`

- Extracts table references from AST's `tableNameList`
- Captures table names and aliases
- Extracts column references for each table
- Returns: `TableNode[]`

#### `extractColumnsForTableFromAST(ast, tableName)`

- Parses select items and column lists from AST
- Filters columns by table reference
- Returns: `string[]` (up to 8 columns)

#### `extractJoinsFromAST(ast)`

- Extracts join relationships from AST's `joinList`
- Captures join type, condition, and source/target tables
- Returns: `JoinEdge[]`

#### `extractCTEsFromAST(ast)`

- Parses CTE definitions from AST
- Extracts CTE body, tables, fields, and usage count
- Analyzes recursion and complexity
- Returns: `CTE[]`

#### `extractMainQueryFieldsFromAST(ast, tables, ctes)`

- Extracts SELECT clause fields from AST
- Maps fields to source tables or CTEs
- Returns: `AnalysisResult['mainQueryFields'][]`

#### `computeMetricsFromAST(ast, ctes, tables, joins)`

- Computes SQL metrics directly from AST:
  - Window functions count
  - GROUP BY and ORDER BY clauses
  - DISTINCT usage
  - Subquery depth
  - Join and CTE counts
- Returns: `SqlMetrics`

#### `computeSubqueryDepthFromAST(ast)`

- Traverses AST recursively to calculate subquery nesting depth
- Returns: `number`

## Error Handling

The implementation includes robust error handling:

1. **Parser Initialization**: Wrapped in try-catch block
2. **AST Validation**: Checks for `ast` and `ast.ast` existence
3. **Fallback Mechanism**: Automatically uses regex-based extraction if:
   - Parser fails
   - AST is invalid or empty
   - Any extraction function throws

4. **Console Warnings**: Logs parser errors for debugging
